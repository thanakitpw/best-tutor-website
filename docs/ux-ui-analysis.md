## 1. Hero Section — ปรับเพื่อ Lead

### ปัญหา
- แบนเนอร์ + ปุ่ม CTA "หาครูสอนพิเศษ" เฉย ๆ ไม่ดึงดูด
- ไม่มีตัวเลขสถิติสร้างความน่าเชื่อถือ
- ต้องคลิกไปหน้าอื่นเพื่อเริ่มหาครู

### ควรปรับเป็น
```
┌─────────────────────────────────────────────────────────┐
│  หาครูสอนพิเศษที่ใช่ ใกล้บ้านคุณ                        │
│  "เป้าหมายของคุณ ความสำเร็จของเรา"                      │
│                                                          │
│  ┌─────────────┬──────────────┬──────────┐              │
│  │ เลือกวิชา ▼ │ เลือกพื้นที่ ▼ │ [ค้นหา] │              │
│  └─────────────┴──────────────┴──────────┘              │
│                                                          │
│  ✓ 30,000+ นักเรียน  ✓ 500+ ติวเตอร์  ✓ 4.8 คะแนนเฉลี่ย │
└─────────────────────────────────────────────────────────┘
```

**เหตุผล:**
- Search Bar ตรง Hero = ลด friction จาก 2 คลิก เหลือ 0 คลิก
- Stats Bar = Social Proof ทันทีที่เห็น
- ทุก Marketplace ที่ประสบความสำเร็จ (Airbnb, Wongnai, FindMyTutor) ใช้ pattern นี้

## 2. ระบบค้นหา/Filter — เพิ่ม Lead ได้มากที่สุด

### ปัญหา
- ไม่มีระบบค้นหาจริง มีแค่ฟอร์ม lead-capture 5 ขั้นตอน
- ผู้ใช้ต้องให้เบอร์โทรก่อนเห็นติวเตอร์ = Friction สูงมาก
- ไม่มีทางรู้ว่ามีติวเตอร์กี่คนในวิชาที่ต้องการ

### ควรปรับเป็น 2 ทางเลือก

**ทางเลือก A: ค้นหาด้วยตัวเอง (เพิ่มใหม่)**
```
[Filter Sidebar]           [ผลลัพธ์]
┌──────────────┐          ┌──────────────────┐
│ วิชา: ▼      │          │ พบ 24 ติวเตอร์    │
│ พื้นที่: ▼    │          │                    │
│ ช่วงราคา: ▼  │          │ [การ์ดติวเตอร์ 1]  │
│ คะแนนรีวิว: ▼│          │ [การ์ดติวเตอร์ 2]  │
│ เรียงตาม: ▼  │          │ [การ์ดติวเตอร์ 3]  │
│              │          │ ...                │
│ [ล้าง] [ค้นหา]│          │                    │
└──────────────┘          └──────────────────┘
```
- เห็นติวเตอร์ทันทีไม่ต้องให้ข้อมูลส่วนตัว
- Filter: วิชา, พื้นที่, ช่วงราคา, คะแนนรีวิว, ประสบการณ์
- คลิกดูโปรไฟล์ → ค่อยกดติดต่อ/กรอกฟอร์ม

**ทางเลือก B: ให้ระบบจับคู่ (ปรับจากฟอร์มเดิม)**
```
ขั้นตอน 1: เลือกวิชา + ระดับชั้น
ขั้นตอน 2: เลือกพื้นที่ + เป้าหมาย
→ แสดงติวเตอร์ที่เหมาะ 3-5 คน (ไม่ต้อง login)
→ กดสนใจ → ค่อยกรอกชื่อ + เบอร์โทร
```

**หลักการ: ให้เห็นคุณค่าก่อน ค่อยขอข้อมูล (Value Before Ask)**

## 5. โครงสร้าง Homepage ใหม่ (เน้น Lead + SEO)

### เว็บเดิม
```
Hero → เลือกหลักสูตร (5 ไอคอน) → คอร์สแนะนำ (4 การ์ด) → ติวเตอร์ยอดนิยม → Footer
```

### ควรปรับเป็น
```
1. Hero + Search Bar + Stats (Lead: ค้นหาทันที)
2. หมวดวิชายอดนิยม + จำนวนติวเตอร์ (SEO: internal links)
3. ติวเตอร์แนะนำ พร้อมรีวิว (Lead: social proof)
4. "ทำงานอย่างไร" 3 ขั้นตอน (Lead: ลด friction)
5. รีวิวจากนักเรียน — Testimonials (Lead: trust)
6. คอร์สแนะนำ (SEO: เนื้อหาเพิ่มเติม)
7. FAQ — คำถามที่พบบ่อย (SEO: featured snippets)
8. CTA "สมัครเป็นติวเตอร์" (Lead: ดึงดูดติวเตอร์ใหม่)
9. โลโก้พาร์ทเนอร์/มหาวิทยาลัย (Lead: trust)
10. Footer + Sitemap links (SEO: internal linking)
```

**Section ใหม่ที่เพิ่ม:**
- **"ทำงานอย่างไร"** — แสดง 3 ขั้นตอนง่าย ๆ: เลือกวิชา → เลือกครู → เริ่มเรียน
- **FAQ** — คำถามที่คนค้นหาบ่อย เช่น "เรียนพิเศษราคาเท่าไหร่" "หาครูสอนพิเศษยังไง"
- **Testimonials** — รีวิวจริงจากนักเรียน/ผู้ปกครอง

## 6. ฟอร์มหาครูสอนพิเศษ — ลด Friction เพื่อเพิ่ม Lead

### เว็บเดิม (5 ขั้นตอน)
```
1. เลือกวิชา → 2. เป้าหมาย → 3. ระดับชั้น → 4. พื้นที่ → 5. ชื่อ+เบอร์+Email+Line
```
ปัญหา: ขอเบอร์โทรก่อนเห็นอะไร = Drop-off สูงมาก

### ควรปรับเป็น (2 ขั้นตอน + ผลลัพธ์)
```
ขั้นตอน 1: เลือกวิชา + ระดับชั้น + พื้นที่ (3 dropdown)
    ↓
แสดงผล: "พบติวเตอร์ 12 คนที่เหมาะกับคุณ" + แสดงการ์ด 3-5 คน
    ↓
ขั้นตอน 2: กรอกชื่อ + เบอร์โทร (แค่ 2 ช่อง) เพื่อนัดเรียน
```

**หลักการ:**
- แสดงคุณค่าก่อนขอข้อมูล
- ลดจำนวนฟิลด์ที่ required (ชื่อ + เบอร์โทร เท่านั้น)
- Email + Line ID เป็น optional

## 7. SEO — สิ่งที่ต้องเพิ่มในทุกหน้า

### 7.1 Schema Markup (JSON-LD)

**หน้าแรก:**
```json
{
  "@type": "Organization",
  "name": "Best Tutor Thailand",
  "aggregateRating": { "ratingValue": "4.8", "reviewCount": "500" }
}
```

**หน้าโปรไฟล์ติวเตอร์:**
```json
{
  "@type": "Person",
  "name": "ครูเนย ชวัลญา",
  "jobTitle": "ติวเตอร์ภาษาอังกฤษ",
  "aggregateRating": { "ratingValue": "4.9", "reviewCount": "28" },
  "review": [...]
}
```

**หน้าบทความ:**
```json
{
  "@type": "Article",
  "headline": "...",
  "author": { "@type": "Organization", "name": "Best Tutor Thailand" }
}
```

**FAQ:**
```json
{
  "@type": "FAQPage",
  "mainEntity": [{ "@type": "Question", "name": "...", "acceptedAnswer": {...} }]
}
```

### 7.2 Meta Tags ต่อหน้า

| หน้า | Title Pattern | Description Pattern |
|---|---|---|
| หน้าแรก | Best Tutor Thailand - หาครูสอนพิเศษตัวต่อตัว | หาครูสอนพิเศษที่ใช่ ใกล้บ้านคุณ ติวเตอร์คุณภาพกว่า 500 คน... |
| หมวดวิชา | ติวเตอร์[วิชา] - หาครูสอน[วิชา]ตัวต่อตัว \| Best Tutor | รวมติวเตอร์[วิชา]คุณภาพ [N] คน สอนตัวต่อตัว ทั้งออนไลน์และ... |
| โปรไฟล์ติวเตอร์ | ครู[ชื่อ] - ติวเตอร์[วิชา] ★[คะแนน] \| Best Tutor | ครู[ชื่อ] ติวเตอร์[วิชา] จาก[มหาวิทยาลัย] ประสบการณ์ [N] ปี... |
| บทความ | [หัวข้อบทความ] \| Best Tutor Thailand | [excerpt 155 ตัวอักษร] |

### 7.3 Internal Linking Strategy

- ทุกหน้าวิชา → link ไปหน้าวิชาย่อย + ติวเตอร์ในวิชานั้น
- ทุกโปรไฟล์ติวเตอร์ → link ไปหน้าวิชาที่สอน + ติวเตอร์คนอื่นในวิชาเดียวกัน
- ทุกบทความ → link ไปหน้าวิชาที่เกี่ยวข้อง + ติวเตอร์ที่แนะนำ
- Footer → Sitemap links ทุกหมวดวิชา

### 7.4 Programmatic SEO

สร้างหน้า SEO อัตโนมัติจากข้อมูลที่มี:
- `/subject/[วิชา]/` → "ติวเตอร์[วิชา] - รวมครูสอน[วิชา]คุณภาพ"
- `/tutor/[ชื่อ]/` → "ครู[ชื่อ] ติวเตอร์[วิชา] [พื้นที่]"
- `/blog/[หมวด]/` → บทความตามหมวดวิชา

## 8. Navigation & Header

### ปัญหา
- ไม่มี Search ใน Navbar
- ไม่มี Sticky navbar
- Mega dropdown วิชารกเกินไป
- ไม่มีเบอร์โทร/Line ใน header (mobile)

### ควรปรับเป็น
```
┌──────────────────────────────────────────────────┐
│ [Logo]  หน้าแรก  รายวิชา▼  บทความ  [🔍]  [หาครู] │
└──────────────────────────────────────────────────┘
```

- **Floating sticky navbar** — ซ่อนตอน scroll ลง แสดงตอน scroll ขึ้น
- **Search icon** ที่กดแล้วขยายเป็น search bar
- **Dropdown วิชา** แสดงเป็นหมวดหลัก + ไอคอน (ไม่ต้องแสดงทุกวิชาย่อย)
- **ปุ่ม CTA "หาครู"** สีส้มโดดเด่น
- **Mobile:** hamburger menu + ปุ่ม Line/โทร

## 9. Mobile UX — ผู้ใช้ส่วนใหญ่มาจาก Mobile

### ปัญหา
- Touch target เล็กเกินไป (ต่ำกว่า 44x44px)
- ฟอร์ม 5 ขั้นตอนบน mobile ใช้ยาก
- ไม่มี Floating CTA
- การ์ดติวเตอร์ไม่ optimize สำหรับ mobile

### ควรปรับ
- ทุกปุ่ม/ลิงก์ต้อง **44x44px ขึ้นไป**
- ฟอร์มหาครู **ลดเหลือ 2 ขั้นตอน** (ดูข้อ 6)
- เพิ่ม **Floating CTA** มุมขวาล่าง: "หาครู" หรือ "Line"
- การ์ดติวเตอร์บน mobile: **horizontal scroll** หรือ **stacked list**
- **Bottom navigation bar** สำหรับ mobile: หน้าแรก | ค้นหา | วิชา | บทความ | เมนู

## 10. ฟอร์มสมัครติวเตอร์ — 22 ฟิลด์ลดเหลือ 3 ขั้นตอน

### เว็บเดิม
22 ฟิลด์ในหน้าเดียว ไม่มี progress bar ไม่มี save draft

### ควรปรับเป็น 3 ขั้นตอน
```
ขั้นตอน 1/3: ข้อมูลพื้นฐาน
├── ชื่อ-นามสกุล, ชื่อเล่น, เพศ
├── Email, เบอร์โทร, Line ID
└── อาชีพปัจจุบัน

ขั้นตอน 2/3: ข้อมูลการสอน
├── วุฒิการศึกษา
├── ประสบการณ์สอน (ปี)
├── วิชาที่สอน (multi-select)
├── แนวการสอน
├── ราคา/ชม.
└── พื้นที่สอน + มีรถยนต์ไหม
```

## 11. Design System ใหม่

### Color Palette (เน้น Lead Conversion)
```
Primary:     #3B82F6  (น้ำเงินสด — น่าเชื่อถือ, การศึกษา)
Secondary:   #60A5FA  (น้ำเงินอ่อน)
CTA:         #F97316  (ส้ม — ดึงดูดสายตา, กดปุ่ม)
CTA Hover:   #EA580C  (ส้มเข้ม)
Success:     #22C55E  (เขียว — ยืนยัน, สำเร็จ)
Warning:     #F59E0B  (เหลือง)
Error:       #EF4444  (แดง)
Background:  #FFFFFF  (ขาว)
Alt BG:      #F8FAFC  (เทาอ่อนมาก — สลับ section)
Text:        #1E293B  (เทาเข้ม)
Muted:       #64748B  (เทากลาง)
Border:      #E2E8F0  (เทาอ่อน)
Star:        #FBBF24  (ทอง — คะแนนดาว)
```

**เหตุผลเลือกส้มเป็น CTA:**
- ตัดกับน้ำเงินได้ดี (complementary color)
- สะดุดตา ดึงดูดการคลิก
- ใช้กันแพร่หลายในเว็บ Marketplace

### Typography (รองรับภาษาไทย)
```
Heading (EN): Plus Jakarta Sans — font-weight 600-700
Heading (TH): Noto Sans Thai — font-weight 600-700
Body:         Noto Sans Thai + Plus Jakarta Sans — font-weight 400
```

**Tailwind Config:**
```js
fontFamily: {
  sans: ['Plus Jakarta Sans', 'Noto Sans Thai', 'sans-serif'],
  heading: ['Plus Jakarta Sans', 'Noto Sans Thai', 'sans-serif'],
}
```

**Font Scale:**
```
H1: 36px / 2.25rem — หัวข้อหลัก
H2: 30px / 1.875rem — หัวข้อ section
H3: 24px / 1.5rem — หัวข้อย่อย
H4: 20px / 1.25rem — หัวข้อการ์ด
Body: 16px / 1rem — เนื้อหาหลัก
Small: 14px / 0.875rem — ข้อมูลเสริม
Caption: 12px / 0.75rem — label, badge
```

### Layout
```
Max Width:    1280px (container)
Grid:         12 columns
Gap:          24px (default), 48px (section)
Border Radius: 8px (cards), 12px (large cards), 9999px (badges/pills)
Shadow:       sm (cards), md (hover), lg (modals)
```

### Responsive Breakpoints
```
Mobile:  < 640px  (sm)
Tablet:  640-1024px (md)
Desktop: > 1024px (lg)
Wide:    > 1280px (xl)
```
